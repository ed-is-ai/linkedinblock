## Phase 10: Profile Insights

**Goal**: The dashboard shows what percentage of all unique author profiles seen in the feed were flagged as bots, not just what percentage of posts were hidden  
**Depends on**: Phase 6  
**Requirements**: INSIGHT-01, INSIGHT-02  
**Success Criteria:**

1. Each unique author profile seen by the content script is counted once per calendar day in `dailyStats` — a prolific poster increments the counter by 1 regardless of how many posts they publish that day
2. The dashboard displays a "Profile bot rate" figure: the percentage of unique profiles seen in the selected time window (7 or 30 days) that are flagged accounts
3. The new stat is visible on the dashboard without requiring any popup changes

## Phase 11: Posts CSV Export

**Goal**: Users can download their stored hidden post data as a CSV for analysis in a spreadsheet  
**Depends on**: Phase 9  
**Requirements**: EXPORT-03  
**Success Criteria:**

1. Clicking "Export Posts CSV" in the dashboard downloads `linkedin-blocker-posts-YYYY-MM-DD.csv`
2. The file has one row per stored post with columns: `authorId, authorName, urn, score, text, hiddenAt`
3. All text fields (especially post `text`) are correctly RFC 4180-escaped so the file opens without corruption in Excel / Google Sheets

## Key Design Decisions

| Decision | Outcome |
|----------|---------|
| Profile deduplication | `seenProfileIds: string[]` embedded in each `DailyStats` entry — one ID per author per day |
| Profile bot-rate denominator | Union Set of all `seenProfileIds` across window days → truly unique profiles seen in window |
| Bot count for rate | Non-dismissed flagged accounts whose `authorId` appears in the union |
| Flush gate | `seenProfileIds` written only on hide events (consistent with existing `seenToday` / `hiddenToday` behaviour) |
| Posts CSV columns | `authorId, authorName, urn, score, text, hiddenAt` — RFC 4180, `hiddenAt` as ISO-8601 |
| Posts CSV button location | "Data management" card alongside existing Export JSON / Export CSV |
