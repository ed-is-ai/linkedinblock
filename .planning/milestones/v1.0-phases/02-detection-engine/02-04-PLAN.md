## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LinkedIn DOM -> postNode.classList | Our code writes class names onto LinkedIn-owned nodes |
| Content script -> chrome.storage.local | Detection metadata persists across sessions |
| Content script -> service worker | POST_HIDDEN message crosses extension contexts |
| Service worker -> chrome.action | Badge text/colour updates affect browser UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-14 | Information Disclosure | persistFlaggedAccount storage write | mitigate | FlaggedAccountStub explicitly omits postText. The persistFlaggedAccount helper signature has no postText parameter so it cannot accidentally flow into storage. Verified via grep gate in Task 2 |
| T-02-15 | Tampering | Direct chrome.storage.local calls bypass typed wrapper | mitigate | Task 2 verify grep gate forbids `chrome.storage.local.` literal in content/index.ts. INFRA-03 enforces wrapper-only access |
| T-02-16 | Denial of Service | Hide breaks LinkedIn infinite scroll sentinel | mitigate | `.llb-hidden` applied to post card only (not parent). Human verification step 9 specifically tests this. Pitfall 3 escape valve: if infinite scroll breaks, revert to tombstone-only (no display:none) |
| T-02-17 | Denial of Service | Extension context invalidated on reload | mitigate | sendMessage wrapped in .catch() per PITFALLS COMMON-4. detector.detect promise also has a .catch() that logs and continues |
| T-02-18 | Repudiation/ToS | Programmatic clicks on LinkedIn UI elements | mitigate | Only comment-expand button is clicked (read-only per D-02); cap of 10 per page load. Block/follow actions are NEVER clicked - those use deep links in Phase 5 per CLAUDE.md constraint #3 |
| T-02-19 | Spoofing | Non-typed inbound message from content script | mitigate | SW handler checks `message?.type === 'POST_HIDDEN'` with optional chaining; any other shape returns false with no action |
| T-02-SC | Tampering | npm/pip/cargo installs in this plan | accept | No package installs in Plan 04 (TypeScript edits only). Plan 02 already passed the Package Legitimacy Gate for fast-levenshtein |
