    Internals must include four named (non-exported) helpers:

    1. `waitForFeedContainer(): Promise<Element | null>` — loops up to `MAX_RETRIES = 10` with delay `BASE_DELAY_MS * Math.min(attempt + 1, 4)` where `BASE_DELAY_MS = 500`. Tries `document.querySelector(FEED_CONTAINER)` first, then `document.querySelector(FEED_CONTAINER_FALLBACK)`. On final failure, emit health sentinel: `console.warn('[LLB] Feed container not found after retries.', { selectorsVersion: SELECTORS_VERSION, url: location.href })` and return `null`.

    2. `attachObserver(container: Element, onPost): MutationObserver` — creates a `MutationObserver` with handler iterating `mutations`, skipping any record whose `type !== 'childList'`, and for each `addedNode` of `nodeType === Node.ELEMENT_NODE`, computes `postCards = el.matches(POST_CARD) ? [el] : Array.from(el.querySelectorAll(POST_CARD))`. For each card, reads `urn = card.getAttribute(POST_URN_ATTR)`; if falsy or already in the module-level `processedPosts: Set<string>`, continue; otherwise add to set, extract `authorName` by querying `POST_AUTHOR_NAME` within the card (`card.querySelector(POST_AUTHOR_NAME)?.textContent?.trim() ?? '<unknown>'`), then call `onPost({ urn, authorName })`. Returns the observer with `observer.observe(container, { childList: true, subtree: true })`. Do NOT include `attributes: true` or `characterData: true`.

    3. `installSpaNavigationHandler(reinit: () =&gt; void): void` — exactly per RESEARCH.md Pattern 3: maintains module-scope `lastUrl`, wraps `history.pushState` with `originalPushState.bind(history)` then delegates, on URL change calls `setTimeout(reinit, 1000)`. Adds `window.addEventListener('popstate', ...)` with the same URL-change guard.

    4. `reinit(): Promise<void>` — disconnects current observer (held in a module-scope `let currentObserver: MutationObserver | null`), clears `processedPosts`, awaits `waitForFeedContainer()`, and re-runs `attachObserver` if a container is found.

    `startObserving(onPost)` orchestrates: calls `reinit()` once on startup with the supplied `onPost`, then calls `installSpaNavigationHandler(() =&gt; reinit())`. Store `onPost` at module scope so `reinit` can re-bind it on each route change.

    All selector imports must come from `./selectors`. There must be ZERO occurrences of the strings `data-urn`, `data-finite-scroll`, `data-anonymize`, `aria-label` in this file as raw string literals — those values live only in `selectors.ts` per INFRA-04.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LinkedIn DOM → content script callback | Untrusted DOM text crosses into `onPost` callback (URN string, author textContent) |
| Content script → page console.log | URN and author name logged to console — visible to anyone with DevTools access to that LinkedIn tab |
| history.pushState monkey-patch → page JavaScript | Wrapped function still callable by LinkedIn's own code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | history.pushState wrapper | mitigate | Wrapper calls `originalPushState.bind(history)(...args)` BEFORE the URL-change check, so LinkedIn's navigation is never blocked or modified; only post-hoc detection is added |
| T-04-02 | Information Disclosure | Console-logged author names | accept | Author names are public LinkedIn profile data visible in the feed; logged only to the user's own DevTools console; no remote transport. Will be removed when scoring lands in Phase 2 |
| T-04-03 | Denial of Service | MutationObserver on document.body subtree | mitigate | Observer attached to feed container (not document) with options `{childList:true, subtree:true}` only — attributes/characterData explicitly excluded (RESEARCH.md anti-pattern) |
| T-04-04 | Denial of Service | processedPosts Set unbounded growth | accept | Set is cleared on every SPA navigation (CONTEXT.md decision); a single feed session of ~hundreds of URNs is well within memory bounds. LRU eviction deferred to Phase 2/3 per CONTEXT.md Deferred Ideas |
| T-04-05 | Tampering | Content script in MAIN world | mitigate | manifest.json content_scripts entry omits `"world"` — defaults to ISOLATED. No access to LinkedIn's page JS scope or globals |
| T-04-06 | Information Disclosure | Health sentinel exposing selector internals | accept | console.warn includes `SELECTORS_VERSION` and `location.href` — both already visible to anyone inspecting the extension source; no sensitive data |
</threat_model>
