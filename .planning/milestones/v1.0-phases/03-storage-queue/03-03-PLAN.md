    ```typescript
    /** EMA smoothing factor for composite score updates. Lower = slower to change; higher = more reactive to new posts. */
    const EMA_ALPHA = 0.2;

    /** Maximum number of entries in flaggedAccounts before eviction triggers. */
    const QUEUE_CAP = 500;
    ```

    **Exported function signature:**
    ```typescript
    export async function persistFlaggedAccount(opts: {
      authorId: string;
      authorName: string;
      authorProfileUrl: string;
      compositeScore: number;
      signals: Record<string, number>;
      hiddenPostUrn: string | null;
    }): Promise<void>
    ```

    **Logic (implement in this order):**

    1. Read `flaggedAccounts` from storage: `const { flaggedAccounts = {} } = await storageGet(['flaggedAccounts']);`

    2. Get existing entry: `const existing = flaggedAccounts[authorId];`

    3. `const now = Date.now();`

    4. **Update path** (existing entry found):
       - EMA: `existing.compositeScore = existing.compositeScore * (1 - EMA_ALPHA) + compositeScore * EMA_ALPHA;`
       - Peak: `existing.peakScore = Math.max(existing.peakScore, compositeScore);`
       - Migration for Phase 2 entries without postCount: `existing.postCount = (existing.postCount ?? 0) + 1;`
         (The `?? 0` handles entries written by Phase 2 code before postCount existed — D-04)
       - Update timestamps and name: `existing.lastSeenAt = now; existing.authorName = authorName; existing.authorProfileUrl = authorProfileUrl;`
       - Merge signals (accumulate with Math.max per signal key, so the highest observed per-signal score is retained):
         ```typescript
         for (const [key, val] of Object.entries(signals)) {
           existing.signals[key] = Math.max(existing.signals[key] ?? 0, val);
         }
         ```
       - Append hidden URN if provided and not already present:
         ```typescript
         if (hiddenPostUrn !== null && !existing.hiddenPostUrns.includes(hiddenPostUrn)) {
           existing.hiddenPostUrns.push(hiddenPostUrn);
         }
         ```
       - `flaggedAccounts[authorId] = existing;`

    5. **Create path** (new entry — NOT in existing flaggedAccounts):
       a. **Eviction check** (D-12, D-13): if `Object.keys(flaggedAccounts).length >= QUEUE_CAP`, evict before inserting:
          - Find the entry with the lowest `compositeScore`. On tie, evict the one with the smallest `firstSeenAt` (oldest).
          - Use a simple reduce over Object.entries to find the evict key:
            ```typescript
            const evictId = Object.entries(flaggedAccounts).reduce((worst, [id, entry]) => {
              const w = flaggedAccounts[worst];
              if (entry.compositeScore < w.compositeScore) return id;
              if (entry.compositeScore === w.compositeScore && entry.firstSeenAt < w.firstSeenAt) return id;
              return worst;
            }, Object.keys(flaggedAccounts)[0]);
            delete flaggedAccounts[evictId];
            ```
       b. Create the new `FlaggedAccount` entry:
          ```typescript
          const entry: FlaggedAccount = {
            authorId,
            authorName,
            authorProfileUrl,
            compositeScore,
            peakScore: compositeScore,
            postCount: 1,
            signals: { ...signals },
            hiddenPostUrns: hiddenPostUrn !== null ? [hiddenPostUrn] : [],
            firstSeenAt: now,
            lastSeenAt: now,
            status: 'pending',
          };
          flaggedAccounts[authorId] = entry;
          ```

    6. Write back: `await storageSet({ flaggedAccounts });`

    **Imports needed:**
    ```typescript

    import type { FlaggedAccount } from './types';
    ```

    The file must not import from `content/` — it is a shared module. It must not reference `document.` or `chrome.runtime.`. Only `chrome.storage.local` (via the storage wrappers).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-Q-01 | Tampering | Eviction of legitimate entries | accept | Eviction only fires at 500 entries; lowest-score entry is least suspicious; by definition the lowest-risk entry to lose |
| T-03-Q-02 | Information Disclosure | dismissedAccounts contains authorId slugs | accept | Profile slugs are public information; array stored in local storage only, never transmitted |
| T-03-Q-03 | Denial of Service | Large profileSignalCache in memory | accept | Capped implicitly by session post count; cleared on SPA navigation; Map of string→Record<string,number> is negligible memory |
