## Feed Insights

- [x] **INSIGHT-01**: Each unique author profile seen each day is recorded in `dailyStats.seenProfileIds: string[]` — an author whose posts appear multiple times on the same day is stored only once in that day's array
- [x] **INSIGHT-02**: The dashboard shows a "Profile bot rate" stat: the percentage of unique profiles seen in the selected time window (union of seenProfileIds across window days) that are flagged accounts; displayed alongside existing post-level stats

## Export

- [x] **EXPORT-03**: User can export all stored hidden posts as a separate CSV file (`linkedin-blocker-posts-YYYY-MM-DD.csv`) with columns: `authorId, authorName, urn, score, text, hiddenAt`

## Noted Limitation

INSIGHT-01/02: `seenProfileIds` only flushed to storage when at least one post is hidden that day (same gate as `seenToday`). On days where the feed is browsed but nothing is hidden, profile data is not persisted. Consistent with pre-existing design.
