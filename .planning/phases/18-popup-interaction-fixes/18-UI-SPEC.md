## Interaction Contract

| Interaction | Trigger | Result | Notes |
|-------------|---------|--------|-------|
| Click account name | `onClick` on `<a>` element | Opens LinkedIn profile in new tab | `e.stopPropagation()` added — does NOT expand row |
| Click account row (outside name link) | `onClick` on `summaryArea` div | Toggles expand/collapse | Unchanged behaviour |
| Click Block button | `onClick` on Block button | Writes `status: 'blocked'` to storage; account moves to Blocked section | No navigation — window.open removed |
| Click Blocked section header | `onClick` on header div | Toggles `blockedExpanded` state | Collapsed by default |
| Storage `flaggedAccounts` onChanged | Chrome storage listener | Popup re-renders; moved account appears in Blocked section | Existing listener — no new wiring needed |

## States and Conditions

| Condition | Visual Result |
|-----------|--------------|
| No pending accounts | Empty state paragraph shown; no Blocked section visible if also no blocked accounts |
| Pending accounts exist | List rendered sorted by peakScore desc |
| Blocked accounts exist | Collapsed "Blocked (N) ▾" header shown below pending list |
| Blocked section expanded | AccountRow list with `isBlocked=true` rows rendered |
| Account row `isBlocked=true` | Grey name, "Blocked" chip, no Block/Dismiss buttons |
| Account row `isBlocked=false` | Normal name colour, Block + Dismiss buttons present |
