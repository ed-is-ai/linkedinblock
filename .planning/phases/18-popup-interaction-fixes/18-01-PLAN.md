<objective>
Fix BUG-01: the content script hides posts from manually-blocked authors at init but has no equivalent path for accounts whose stored `peakScore` already meets the auto-hide threshold from prior sessions. Add a `thresholdAuthors` map that mirrors the existing `blockedAuthors` pattern — populated at `init()` and kept fresh by `chrome.storage.onChanged` — so threshold-crossing accounts are hidden immediately on load and on new infinite-scroll posts, without re-running the async detector.

Purpose: Stored `peakScore` is authoritative (D-01); re-scoring on every load would cause a visible delay and waste detector calls. This closes the gap where a user who set a threshold in a prior session still sees those posts on reload.
Output: Updated `src/content/index.ts` with a `thresholdAuthors` map, a `currentThreshold` module variable, an observer hide branch, and a `settings` onChanged rebuild branch.
</objective>
