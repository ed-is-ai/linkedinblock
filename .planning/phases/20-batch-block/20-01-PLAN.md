|----------|-------------|
| user click → popup state | Trusted local UI interaction; the only state-changing trigger in this phase |
| popup → chrome.storage.local | Same-origin extension storage; no network, no cross-context message |

This phase is popup-only with no network calls, no message passing to the service worker or content script, and no user-supplied free-text input — the sole action is a confirmed button click that mutates a known set of local-storage records.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01 | Tampering (accidental) | handleBatchBlock bulk write | mitigate | Inline confirmation gate (D-05) requires an explicit second click stating the count before any `chrome.storage.local.set`; "Keep pending" leaves storage unchanged. |
| T-20-02 | Tampering (double-submit) | "Block all now" confirm button | mitigate | Button is `disabled` with `cursor: 'not-allowed'` while the async write is in flight (D-07), preventing duplicate/overlapping writes. |
| T-20-03 | Denial of Service (data loss) | flaggedAccounts map write | mitigate | Single batched read-modify-write of the full `flaggedAccounts` map (D-06) preserves all non-qualifying entries; only `status` is changed via spread, no entries deleted. On failure the write rejects atomically and the bar returns to idle (D-08) — no partial corruption. |
| T-20-04 | Information Disclosure | popup data | accept | No new data is read, exposed, or transmitted; all flagged-account data was already present in the popup before this phase. No network surface. |
| T-20-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this phase (uses existing Preact + chrome APIs already in package.json); no Package Legitimacy Gate required. |

No HIGH-severity threats. The confirmation gate (T-20-01) and in-flight disable (T-20-02) cover the accidental/double-write surface; everything else is local-only with no untrusted input.
