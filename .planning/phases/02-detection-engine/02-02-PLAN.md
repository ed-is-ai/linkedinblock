<objective>
Implement the pure-function layer of the heuristic detection engine: five content/engagement signal scorers and one non-English exclusion utility. These are all pure (signals are text-in/number-out; language reads only the `lang` attribute and codepoints from the DOM element passed in). No detector wiring, no DOM mutation, no chrome.* calls — that is Plan 03 and Plan 04. Unit tests live alongside the code so Plan 03 can compose these with confidence.

Purpose: Lock in DETECT-01 (content signals — listicle, buzzword, em-dash, CTA), DETECT-04 (non-English exclusion), DETECT-07 (engagement signal — generic / near-duplicate comments) as deterministic, unit-tested building blocks. Per the Architectural Responsibility Map in 02-RESEARCH.md, scoring is pure text analysis that runs in the content script tier but has zero DOM dependency.

Output:
- fast-levenshtein@3.0.0 + @types/fast-levenshtein@0.0.4 added as direct dependencies
- Five signal modules under `src/content/detector/signals/`, each exporting one named function
- Language detection module `src/content/detector/language.ts`
- A combined Vitest spec under `src/content/detector/signals/__tests__/signals.test.ts` covering happy path + at least one ReDoS-style adversarial input
- A focused spec for isNonEnglish (`language.test.ts`) covering lang attribute, CJK, Cyrillic, Arabic, Latin-only edge cases
</objective>
