<objective>
Two independent changes that complete the dismiss + badge behavior:

**Content script** (`src/content/index.ts`):
- Adds `hiddenPostNodes: Map<string, Element[]>` at module scope to track which DOM nodes were hidden per author
- When a post is hidden, appends `postNode` to this map
- Clears the map on SPA navigation
- Registers a `chrome.storage.onChanged` listener: when `dismissedAccounts` changes, diffs old vs. new to find newly dismissed authors, adds them to `dismissedSet`, removes `llb-hidden` from their tracked nodes, clears their map entry

**Service worker** (`src/background/index.ts`):
- Replaces the `sessionHiddenCount` in-memory counter with a `chrome.storage.onChanged` listener
- When `flaggedAccounts` changes, counts entries with `status === 'pending'` and sets the badge text
- Badge shows empty string when count is 0 (no badge on clean feed)
- Removes the `sessionHiddenCount` variable and its increment from the `POST_HIDDEN` handler (keep the handler itself — content script still sends the message; just don't update badge there anymore)

These two plans (05-01 and 05-02) run in parallel — they touch entirely different files.
</objective>
