<objective>
Wire the Block and Dismiss actions into the popup.

`AccountRow` gains two new callback props — `onBlock` and `onDismiss` — and renders two small buttons at the bottom of each row. The component calls the callbacks on click; all async logic stays in the parent `App`.

`App` provides:
- `handleBlock(account)` — extracts the LinkedIn slug, calls `window.open` with the deep link URL. No storage writes. No `tabs` permission needed.
- `handleDismiss(account)` — reads current `flaggedAccounts` and `dismissedAccounts` from storage, removes the account, appends the authorId, writes both back in one `set` call. The existing Phase 4 `onChanged` listener handles popup refresh automatically — no manual `setAccounts` call needed.
</objective>

         const slug = account.authorProfileUrl.match(/\/in\/([^/?#]+)/)?.[1];
         if (!slug) {
           console.warn('[LLB popup] could not extract slug from', account.authorProfileUrl);
           return;

         );
       }

    2. Add the `handleDismiss` function inside `App` (async):
       ```typescript
       async function handleDismiss(account: FlaggedAccount) {
         const result = await chrome.storage.local.get(['flaggedAccounts', 'dismissedAccounts']);
         const flaggedAccounts = (result.flaggedAccounts ?? {}) as Record<string, FlaggedAccount>;
         const dismissedAccounts = (result.dismissedAccounts ?? []) as string[];

         delete flaggedAccounts[account.authorId];

         const newDismissed = dismissedAccounts.includes(account.authorId)
           ? dismissedAccounts
           : [...dismissedAccounts, account.authorId];

         await chrome.storage.local.set({ flaggedAccounts, dismissedAccounts: newDismissed });
         // Popup state refresh is automatic via the existing onChanged listener (Phase 4)
       }
       ```
